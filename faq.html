What will the old API be deprecated and removed from the browser?

Brandon Jones' Chromium builds have already been officially deprecated.

For Firefox, until we have a stable implementation of the 1.0 API, the existing WebVR API will continue to function. Once 1.0 API is implemented, the old API will no longer function and you will be required to update content to use the new APIs.

When will the new API land in Firefox?

We are hoping to land a stable implementation of the 1.0 APIs into Firefox Nightly within the next quarter. If the API proves stable, it will continue to make its way into Firefox Developer Edition and Firefox Beta in the first half of this year.

The plan is to have these APIs available in Stable, but there is no clear date yet on when that will happen.

Where can I track implementation progress of the new API?

You can track implementation at this Bugzilla bug, and support status on IsWebVRRready.org.

Will old WebVR API be available in release channel?

The existing WebVR API is deprecated and only the new 1.0 API is planned to be available.

Can the new WebVR API be polyfilled?

There are certain features that are possibly only by making changes to the browser platform. The most significant would the ability to traverse links.

When will link traversal links?

Link traversal is planned to be available with the initial release of the 1.0 API implementation.

When will Firefox support room-scale experiences (like Vive)?

Patches have been made to support HTC Vive, but have not landed into the code-base. We do not have a clear timeline for when this will be complete.

Is the spec on the standards body track?

We have formed a W3C Community Group. Standardization of the spec has not begun yet.

How will link traversal work?

With the 1.0 API, content needs to explicitly request content to be displayed into the headset. When navigating to another page by changing window.

Can I use HTML and CSS for WebVR?

This is not currently possible. However, there has been experiments that would enable the copy of DOM element layouts into textures useable in a webGL scene. Though, this will be quite a ways out before we see anything usable by regular web content.

What is WebVR?

WebVR is a set of APIs that offer the ability to render and interact with web content in VR.

Why WebVR?

Most VR content today is being created using proprietary closed-source software. WebVR enables you to build VR experiences using web technologies such as WebGL and utilize the entire bredth of APIs already available in browsers. The Web also offers accessibility by URL, with no download and installs required and completely open platform for distributing your application.

What are the easiest ways to get started building WebVR experiences?

A-Frame, webvr-boilerplate, three.js.

Is WebVR supported on mobile devices?

Firefox for Android currently has WebVR APIs enabled by default. For browsers that do not support WebVR APIs natively, you can polyfill the behavior using the webvr-polyfill.

Which browsers support WebVR?

Currently, Firefox Nightly and experimental builds of Chromium.

Can I use WebVR on Mac or Linux?

The recommended platform for using WebVR is Windows using the Oculus Rift VR headset, or using your mobile device with a Cardboard holder. See section above.

Although using WebVR on Mac OS X and Linux is possible using the Oculus 0.5.0 runtime, it is not recommended. Oculus has suspended support for Mac/Linux as of since Oculus 0.5.0 runtime and it is unclear when support will be resumed.
